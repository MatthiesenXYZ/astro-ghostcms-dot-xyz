## Install

The Best way to install Astro-GhostCMS is from a fresh install.  To get started run the following depending on your package manager of choice.

<Steps>

1. First create a new Astro install with the following command. Then delete the entire `/pages` folder under `/src/`:

   <Tabs>
   <TabItem label="npm" icon="seti:npm">
   ```sh
   npm create astro@latest
   ```
   </TabItem>
   <TabItem label="pnpm" icon="pnpm">
   ```sh
   pnpm create astro@latest
   ```
   </TabItem>
   <TabItem label="yarn" icon="seti:yarn">
   ```sh
   yarn create astro@latest
   ```
   </TabItem>
   </Tabs>

2. Then run the following to install the package!

   <Tabs>
   <TabItem label="npm" icon="seti:npm">
   ```sh title="Astro add method"
   npx astro add @matthiesenxyz/astro-ghostcms
   ```

   ```sh title="Manual pkg Install"
   npm install @matthiesenxyz/astro-ghostcms
   ```
   </TabItem>
   <TabItem label="pnpm" icon="pnpm">
   ```sh title="Astro add method"
   pnpm astro add @matthiesenxyz/astro-ghostcms
   ```
   
   ```sh title="Manual pkg Install"
   pnpm add @matthiesenxyz/astro-ghostcms
   ```
   </TabItem>
   <TabItem label="yarn" icon="seti:yarn">
   ```sh title="Astro add method"
   yarn astro add @matthiesenxyz/astro-ghostcms
   ```
   
   ```sh title="Manual pkg Install"
   yarn add @matthiesenxyz/astro-ghostcms
   ```
   </TabItem>
   </Tabs>

3. Then set your `.env` variables to look like this:

   <Aside> As of v3.3.0 the `CONTENT_API_URL` is no longer required in your `.env` file.  You can set it in your `astro.config.mjs` file as `ghostURL` or as an environment variable.  The `astro.config.mjs` file will take priority. </Aside>

   ```ansi frame="code" title=".env"
   [1;31mCONTENT_API_KEY[0m=[33ma33da3965a3a9fb2c6b3f63b48
   ```

4. Then set your `astro.config.mjs` to look like this:

   ```ts frame="code" title="astro.config.mjs" wrap
   import { defineConfig } from "astro/config";
   import astroGhostCMS from '@matthiesenxyz/astro-ghostcms';

   // https://astro.build/config
   export default defineConfig({
     site: "https://YOUR-DOMAIN-HERE.com"
     integrations: [
       astroGhostCMS({
         // `ghostURL` Recommended to set here, Can also set in .env as CONTENT_API_URL
         ghostURL: "https://ghostdemo.matthiesen.xyz", 
       })
     ],
   });
   ```

5. This is how your file structure should look:

   <FileTree>
   - your-awesome-project/
     - public/
     - src/
     - .env
     - astro.config.mjs
     - package.json
   </FileTree>

6. Then run the following to start your Astro server:

   <Tabs>
   <TabItem label="npm" icon="seti:npm">
   ```sh
   npm run dev
   ```
   </TabItem>
   <TabItem label="pnpm" icon="pnpm">
   ```sh
   pnpm run dev
   ```
   </TabItem>
   <TabItem label="yarn" icon="seti:yarn">
   ```sh
   yarn run dev
   ```
   </TabItem>
   </Tabs>

</Steps>

## Created Routes

The routes are the same as a standard Ghost Blog so you can migrate to Astro easily.

| Route                 | Content                                   |
| --------------------- | ----------------------------------------- |
| `/`                   | Homepage with recents/features Blog Posts |
| `/404`                | 404 Page                                  |
| `/[slug]`             | Post or Page                              |
| `/author/[slug]`      | Author page with related posts            |
| `/authors`            | All the authors                           |
| `/tag[slug]`          | Tag page with related posts               |
| `/tags`               | All the tags                              |
| `/archives/[...page]` | All the posts, paginated                  |
| `/rss.xml`            | All the posts, in a FEED                  |
